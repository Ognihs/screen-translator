# Screen Translator 设计文档

**日期**: 2026-03-29
**状态**: 已确认

## 概述

一个通过截图发送给多模态模型进行翻译的 Python 桌面应用。用户通过鼠标框选屏幕区域，应用按设定间隔自动截取该区域并调用 OpenAI 兼容 API 进行翻译，翻译结果在置顶窗口中显示。

## 技术栈

- **Python 3.14**
- **PySide6** — GUI 框架
- **mss** — 屏幕截图
- **openai** — 多模态 API 客户端（兼容接口）
- **python-dotenv** — 环境变量管理

## 文件结构

```
screen-translator/
├── main.py              # 应用入口，初始化 QApplication
├── config.py            # 配置管理（.env 加载、设置读写）
├── selector.py          # 全屏透明遮罩，鼠标拖拽选区
├── capture.py           # 截图引擎（mss 截取指定区域）
├── translator.py        # OpenAI 兼容 API 客户端
├── border_window.py     # 选区边框提示窗口（置顶、无边框）
├── result_window.py     # 翻译结果展示窗口（置顶、可拖动、可调整大小）
├── control_window.py    # 主控制面板（语言选择、间隔设置、开始/暂停/停止）
├── .env                 # API_KEY, BASE_URL, MODEL 等敏感配置
├── .env.example         # .env 模板
└── pyproject.toml       # 项目依赖
```

## 核心数据流

```
用户点击"选区" → selector.py 全屏遮罩 → 用户拖拽选区 → 返回坐标
                                                          ↓
用户点击"开始" → QTimer 启动 → capture.py 截取选区 → translator.py 调用 API
                                                          ↓
                                              result_window.py 显示翻译结果
```

## 模块详细设计

### config.py — 配置管理

- 使用 `python-dotenv` 加载 `.env` 文件
- 提供的配置项：`API_KEY`、`BASE_URL`、`MODEL`、`DEFAULT_INTERVAL`
- 控制面板中的修改实时生效，不持久化到 `.env`（运行时配置）

### selector.py — 选区模块

- 点击"选择区域"按钮后，创建一个全屏透明窗口（`Qt.FramelessWindowHint` + `Qt.WindowStaysOnTopHint`）
- 窗口背景为半透明黑色遮罩（`rgba(0,0,0,100)`），覆盖所有屏幕
- 鼠标按下拖拽时，绘制蓝色虚线矩形，实时显示选区范围
- 鼠标释放后，返回选区坐标 `(x, y, width, height)`，关闭遮罩窗口
- 支持 ESC 键取消选区

### border_window.py — 选区边框

- 收到选区坐标后，创建一个无边框置顶窗口，精确覆盖在选区边缘
- 窗口内容为空，仅通过 `QPainter` 绘制 2-3px 的红色/绿色边框
- 设置 `Qt.WindowTransparentForInput` 属性，使鼠标事件穿透边框（不影响用户操作下方应用）
- 选区变更时更新位置和大小

### capture.py — 截图引擎

- 接收选区坐标，使用 `mss` 截取该区域
- 返回截图的 `bytes` 数据（PNG 格式），供 API 调用使用
- 纯函数式调用，无状态

### translator.py — 翻译客户端

- 使用 `openai` 库，通过 `base_url` 参数支持兼容接口（DeepSeek、GLM、Ollama 等）
- 将截图以 base64 编码放入消息的 `image_url` 中
- Prompt 模板：`"请将图片中的{source_lang}文本翻译为{target_lang}，仅输出翻译结果，不要添加解释"`
- 超时设置 30 秒，异常时返回错误信息而非崩溃
- 支持的语言：中文、日语、英语（手动选择源语言和目标语言）

### result_window.py — 翻译结果窗口

- `QWidget` 窗口，设置 `Qt.WindowStaysOnTopHint` 置顶
- 标题栏自定义（原生标题栏隐藏），支持拖动
- 右下角拖拽手柄支持调整大小
- 内部使用 `QTextEdit`（只读模式）显示翻译文本
- 支持文本选择和复制
- 每次新翻译到达时替换显示内容

### control_window.py — 控制面板

主界面布局（从上到下）：

```
┌─────────────────────────────┐
│  Screen Translator          │
├─────────────────────────────┤
│ 源语言: [日语 ▼]            │
│ 目标语言: [中文 ▼]          │
│                             │
│ 截图间隔: [10] 秒           │
│                             │
│ API地址: [________________] │
│ 模型:   [________________]  │
│                             │
│ [选择区域]  [开始] [暂停]   │
│            [停止]           │
│                             │
│ 状态: 就绪                  │
└─────────────────────────────┘
```

- 源语言/目标语言：`QComboBox`，选项为中文、日语、英语
- 截图间隔：`QSpinBox`，单位秒，默认 10，范围 1-300
- API 地址、模型名：`QLineEdit`，从 `.env` 读取默认值
- 按钮状态机：**就绪** → 开始 → **运行中** → 暂停/停止 → **就绪**
- 状态栏显示当前状态（就绪/运行中/已暂停/翻译中.../错误）

## 错误处理

### API 调用错误

| 场景 | 处理方式 |
|------|----------|
| API Key 未配置 | 启动时检测，控制面板状态栏显示"请配置 API Key"，禁用"开始"按钮 |
| 网络超时（>30s） | 状态栏显示"翻译超时"，跳过本次，等待下一次定时触发 |
| API 返回错误（4xx/5xx） | 状态栏显示错误信息（如"配额不足"、"模型不可用"），不中断循环 |
| 返回空内容 | 结果窗口显示"（未检测到文本）" |

### 选区相关

| 场景 | 处理方式 |
|------|----------|
| 未选择区域就点击"开始" | 弹出提示"请先选择截图区域"，不启动定时器 |
| 选区面积过小（<10x10px） | 提示"选区过小，请重新选择"，不记录该选区 |
| 显示器分辨率变化 | 不自动处理，用户需重新选择区域 |
| 多显示器 | `mss` 支持多显示器，选区坐标基于虚拟桌面，天然兼容 |

### 定时器与状态管理

| 场景 | 处理方式 |
|------|----------|
| 上一次翻译未完成，定时器又触发 | 跳过本次截图，等下一次定时器触发（不排队） |
| 暂停后恢复 | 清空结果窗口，使用现有选区重新开始计时 |
| 停止 | 清空结果窗口 + 重置选区（隐藏边框，回到未选状态） |
| 停止后重新开始 | 需重新选择区域，否则按"未选择区域"提示处理 |
| 应用关闭时 | 停止定时器，关闭所有窗口，确保无残留进程 |

### 窗口交互

| 场景 | 处理方式 |
|------|----------|
| 结果窗口被用户关闭 | 控制面板状态栏提示"结果窗口已关闭"，下次翻译时自动重新创建 |
| 选区边框被遮挡 | 边框始终置顶，若被其他置顶窗口遮挡则属正常行为 |
| 源语言和目标语言相同 | 允许选择，模型会返回原文，无害 |
